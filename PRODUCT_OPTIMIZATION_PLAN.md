# MathAnim Desktop 产品优化清单 + 开发排期建议

## 1. 文档目的

这份文档用于统一当前阶段的产品优化方向，回答 4 个问题：

1. 这个项目当前最影响体验的瓶颈是什么。
2. 已经完成了哪些高优先级优化。
3. 接下来应该先做什么，后做什么。
4. 如何把"能跑"继续升级成"稳定、清晰、可复盘"的桌面产品。

## 2. 当前产品判断

### 2.1 当前核心问题

项目当前的主要问题，不是"功能太少"，而是"复杂任务一旦失败，用户不知道系统正在做什么，也不知道下一步该怎么处理"。

具体表现为：

- 多步 AI 生成链路一长，渲染失败率明显上升。
- 失败后只能翻长日志，缺少直观诊断。
- 用户难以分辨草稿任务、排队任务、执行中任务、失败任务。
- 保底模式虽然有价值，但以前提示不够明显。
- 左侧操作区和右侧输出区的语义还不够统一。

### 2.2 产品目标

现阶段更合理的目标不是继续堆新功能，而是优先把以下 3 件事做好：

- 过程可见：用户能看到系统当前处于哪一步。
- 失败可懂：失败后能快速知道原因和修补方向。
- 状态清楚：任务的草稿、执行、失败、保底都能一眼识别。

### 2.3 用户高频痛点（新增）

从实际使用角度，用户反馈的高频痛点包括：

- **任务管理**：找历史任务很费劲，没有搜索和筛选，无法批量操作
- **代码编辑**：没有语法高亮，编辑体验像记事本
- **任务复用**：成功案例无法快速复用，没有克隆功能
- **参考图**：只能填 URL，本地图片需要先上传图床
- **配音**：功能藏在设置页，入口不明显
- **预览**：无法调整播放速度，无法导出当前帧

## 3. 已完成优化总览

截至本轮，已经完成一组高优先级优化，重点集中在"复杂任务稳定性"和"前端可理解性"。

### 3.1 后端 / 工作流已完成

- 增加 Manim 失败诊断分类器，能识别 LaTeX、MathTex API、坐标轴标签、3D、布局复杂度等常见失败类型。
- 增强修补提示词策略，让 AI 修补更偏向稳定生成而不是继续冒险。
- 增加保底模式开关，允许在多次失败后切换到简化模板生成。
- 将失败摘要、修补建议、保底状态持久化到任务模型。
- 工作流重试时会带上失败诊断与修补提示，而不是盲修。

### 3.2 前端已完成

当前一共完成了 **10 项前端可见改动**，主要集中在创建页、输出区、任务列表三块。

1. 增加右侧任务日志面板。
2. 增加"保底模式中"状态徽标。
3. 保底任务在任务列表中高亮显示。
4. 输出状态栏改成更易懂的人话状态。
5. 运行中状态支持显示"处理中 / 保底模式中 / 保底排队中"。
6. 草稿任务单独显示为"草稿任务"。
7. 失败任务单独显示为"生成失败"。
8. 失败任务在任务列表中追加红色高亮样式。
9. 输出状态栏和保底徽标支持 hover 诊断 tooltip。
10. 输出区新增固定"失败诊断卡片"，直接展示失败原因和修补建议。

### 3.3 文档已完成

- README 已重写，项目说明更清楚。
- 本优化计划文档已重构为最新版本。

这一部分专门给你复盘"前端到底改了多少、改在哪"。

### 4.1 改动数量

- 前端可见改动：10 项
- 涉及主要文件：3 个

### 4.2 涉及文件

- `src/main/resources/fxml/MainView.fxml`
- `src/main/resources/css/mathanim-theme.css`

说明：

- 严格说 `MainViewController.java` 属于 UI 控制层，不是纯静态前端资源，但它承担了本项目大量界面状态逻辑，因此这里一起纳入前端统计。

### 4.3 用户实际能感知到的变化

- 看到任务时，不再只是"黑箱运行"，而是能知道系统在哪个阶段。
- 看到失败时，不再只剩一大段日志，而是有固定诊断信息。
- 看到保底模式时，不再容易忽略。
- 看到任务列表时，失败任务和保底任务更容易扫到。
- 看到右侧状态栏时，更容易理解当前选中的任务到底是什么状态。

## 5. 当前优先级排序（结合用户反馈更新）

### P0：必须继续做（直接影响可用性）

这些是用户高频场景中最痛的点，优先级最高。

1. **任务搜索和筛选**
   - 历史任务列表支持按概念描述搜索
   - 增加筛选器：全部/草稿/成功/失败/保底/收藏
   - 解决"找之前的成功案例很费劲"的问题

2. **左侧操作区文案统一**
   - 区分"保存草稿""立即生成并执行""执行选中任务""修改后执行"
   - 增加"重试选中任务"和"保底重试"按钮
   - 让状态栏、任务列表、日志标题三处使用统一术语

3. **批量操作能力**
   - 任务列表支持多选（Ctrl+点击）
   - 批量删除失败任务
   - 批量重试、批量导出

4. **代码编辑体验提升**
   - 集成 Python 语法高亮（RichTextFX 或 CodeArea）
   - 增加"格式化代码"按钮
   - 失败时能跳转到错误行

### P1：强烈建议做（明显提升产品感）

这些会让高频用户效率显著提升。

1. **任务标签与收藏**
   - 支持给任务添加自定义标签（力学、微积分、待修改等）
   - 支持标记任务为"收藏"或"模板"
   - 任务列表支持按标签分组

2. **任务克隆和复用**
   - 从历史任务"克隆"创建新任务
   - 支持导出/导入任务配置 JSON
   - 成功任务可标记为"参考案例"

3. **参考图功能增强**
   - 支持拖拽本地图片，自动转 base64
   - 显示参考图缩略图列表
   - 在日志中显示"已加载 X 张参考图"

4. **配音功能入口优化**
   - 在任务列表或预览区增加"为此视频配音"快捷按钮
   - 配音任务和原任务建立关联
   - 增加"预听配音"功能

5. **失败诊断智能化**
   - 解析 Manim 错误日志，提取具体错误行和对象
   - 对常见错误提供"一键修复"建议
   - 记录每次修补的具体改动，形成"修补历史"

### P2：中期增强（产品成熟度）

这些更偏向长期复用和协作场景。

1. **预览功能增强**
   - 增加"导出当前帧为图片"按钮
   - 增加播放速度控制（0.5x、1.5x、2x）
   - 图片预览支持鼠标滚轮缩放

2. **协作和分享能力**
   - 生成"任务报告"（Markdown 或 HTML）
   - 支持分享任务配置给同事
   - 增加"复制视频路径"快捷按钮

3. **智能推荐**
   - 记住用户的常用配置
   - 根据概念描述推荐参考案例
   - 失败任务自动推荐相似的成功案例

4. **性能优化**
   - 任务列表改用虚拟滚动
   - 视频加载时显示加载动画
   - 状态栏显示更细粒度的进度

5. **更多自动化测试**
   - 覆盖 UI 状态逻辑
   - 覆盖失败诊断和保底切换

## 6. 下一阶段开发排期建议（结合用户反馈更新）

下面这个排期结合了当前代码结构、用户高频痛点和实际收益。

### 第 1 周：任务管理核心体验升级

目标：

- 解决"找历史任务很费劲"和"批量操作缺失"两大痛点

建议内容：

- **任务搜索**：在任务列表上方增加搜索框，支持按概念描述、题目内容搜索
- **任务筛选**：增加筛选下拉框（全部/草稿/成功/失败/保底/收藏）
- **批量操作**：任务列表支持 Ctrl+点击多选，增加批量删除按钮
- **操作按钮优化**：统一左侧按钮文案，增加"重试选中任务"和"保底重试"

交付结果：

- 用户能快速找到历史任务
- 批量删除失败任务不再痛苦
- 操作语义更清晰

### 第 2 周：代码编辑和任务复用

目标：

- 提升代码编辑体验，让历史任务变成可复用资产

建议内容：

- **语法高亮**：集成 RichTextFX 或 CodeArea，支持 Python 语法高亮
- **代码格式化**：增加"格式化代码"按钮（调用 black 或 autopep8）
- **任务克隆**：增加"从此任务创建新任务"功能
- **任务标签**：支持给任务添加自定义标签
- **任务收藏**：支持标记任务为收藏

交付结果：

- 代码编辑不再像记事本
- 成功案例可以快速复用
- 任务管理更有条理

### 第 3 周：参考图和配音体验优化

目标：

- 让多模态输入和配音功能更易用

建议内容：

- **参考图拖拽**：支持直接拖拽本地图片，自动转 base64
- **参考图预览**：显示参考图缩略图列表
- **配音快捷入口**：在预览区增加"为此视频配音"按钮
- **配音预听**：混流前可以预听配音效果
- **任务导出/导入**：支持导出任务配置 JSON，方便分享

交付结果：

- 参考图功能更直观
- 配音功能不再藏得太深
- 任务可以分享给同事

### 第 4 周：预览增强和性能优化

目标：

- 提升预览体验和整体性能

建议内容：

- **播放速度控制**：增加 0.5x、1.5x、2x 播放速度选项
- **导出当前帧**：视频预览时可以导出当前帧为图片
- **图片缩放**：图片预览支持鼠标滚轮缩放和平移
- **虚拟滚动**：任务列表改用虚拟滚动，提升大列表性能
- **代码重构**：拆分 `MainViewController` 的状态逻辑

交付结果：

- 预览功能更强大
- 大量任务时不卡顿
- 代码更易维护

如果只继续做一件事，我建议优先做：

也就是在任务列表上方增加：

- 搜索框（按概念描述、题目内容搜索）
- 筛选下拉框（全部/草稿/成功/失败/保底/收藏）

原因很简单：

- 这是用户反馈最强烈的痛点："找之前的成功案例很费劲"
- 这会直接提升高频用户的效率
- 这会让历史任务从"堆积"变成"资产"
- 实现成本相对较低，收益立竿见影
